# GitHub Account Registration and Assessment Repository Setup Guide

Dear students,

For this subject, you will use GitHub to manage and submit your assessment code. Please follow the instructions below carefully.

---

## 1. Create a GitHub Account

Go to:

https://github.com

Create a GitHub account if you do not already have one.

You may use your La Trobe student email address when registering, or you may add it later in your GitHub account settings.

---

## 2. Choose a GitHub Username

Please choose a GitHub username that helps us identify you easily.

Recommended format:

```text
ltu-STUDENTID
```

Example:

```text
ltu-22364751
```

If that username is already taken, you may use another clear version, such as:

```text
ltu-22364751-a
cwa-22364751
22364751-ltu
```

Please avoid using a username that is completely unrelated to your student identity, because it will make assessment management harder.

---

## 3. Create Your Assessment Repository

For each assessment, create a GitHub repository using a clear name.

Recommended repository name for Assessment 1:

```text
CSE3CWA-Assessment-01
```

or

```text
CSE5006-Assessment-01
```

Recommended repository name for Assessment 2:

```text
CSE3CWA-Assessment-02
```

or

```text
CSE5006-Assessment-02
```

---

## 4. Set Your Assessment Repository to Private

Your assessment repository must be set to:

```text
Private
```

Do **not** make your assessment repository public.

This is important because your assessment work should not be visible to other students.

---

## 5. Add the Instructor as a Collaborator

After creating your private assessment repository, please add the instructor as a collaborator.

Instructor GitHub username:

```text
SDinglatrobe
```

### Steps

1. Open your assessment repository on GitHub.
2. Click **Settings**.
3. Click **Collaborators** or **Collaborators and teams**.
4. Click **Add people**.
5. Enter the instructor GitHub username:

```text
SDinglatrobe
```

6. Send the invitation.

Please make sure the invitation is sent before the assessment deadline. Otherwise, your repository may not be accessible for marking.

---

## 6. Check Before Submission

Before the deadline, please check that:

* Your repository is **private**.
* Your latest code/files have been pushed to GitHub.
* Your repository name is clear.
* `SDinglatrobe` has been added as a collaborator.

---

## 7. Submit Your GitHub Information in LMS

When you submit your assessment in LMS, please include the following information in the LMS submission text box and then click **Submit**:

```text
Student ID:
La Trobe email:
First name:
Last name:
GitHub username:
Assessment repository URL:
```

Example:

```text
Student ID: 22364751
La Trobe email: 22364751@students.latrobe.edu.au
First name: Alex
Last name: Smith
GitHub username: ltu-22364751
Assessment repository URL: https://github.com/ltu-22364751/CSE3CWA-Assessment-01
```

This information is required so that your GitHub repository can be correctly matched with your student record.

Please make sure the repository URL is the URL of **your own private assessment repository**, not the weekly teaching code repository.

---

## Important Notes

* If you change your GitHub username after submitting it, you must notify the teaching team immediately.
* After the submission deadline, please do **not** modify your GitHub repository.
* Your assessment repository may be downloaded and marked shortly after the deadline. Any changes made after the deadline may not be considered during marking.
* If you have special circumstances or need to make changes after the deadline, please contact your lecturer as soon as possible.
